MOTODASH

Upload Manual ke GitHub — 100% Lewat Browser, Tanpa Aplikasi Tambahan

✅ Tidak perlu Git · Tidak perlu Terminal · Tidak perlu VS Code

📌 Cara Kerja Tutorial Ini

Semua dilakukan lewat File Explorer (bawaan Windows) dan browser (Chrome/Edge) yang sudah pasti ada di komputer Anda. Tidak ada software tambahan yang perlu diinstall untuk meng-upload kode ke GitHub.

Fase A — Siapkan File di Komputer
A1

Extract File ZIP

1
Download motodash_final.zip dari Claude

Simpan di lokasi mudah, contoh Desktop atau C:\

2
Klik kanan file ZIP → "Extract All..."

Pilih lokasi extract, contoh: C:\motodash\

3
Pastikan struktur folder seperti ini:
motodash\ ├── index.html ├── manifest.json ├── sw.js ├── generate-icons.html ├── README.md CARA_PAKAI.html dll ├── css\ ├── js\ ├── assets\icons\ └── vendor\ ← folder penting, JANGAN dilewat
A2

Generate Icon PNG — Double-Click Saja

1
Masuk folder motodash → double-click generate-icons.html

Browser terbuka otomatis menampilkan halaman generator

2
Klik tombol "🎨 Generate Icon Sekarang"

Dua preview icon muncul dalam 1-2 detik

3
Klik download kedua file

⬇ icon-192.png   ⬇ icon-512.png

4
Pindahkan kedua file PNG ke folder:
C:\motodash\assets\icons\

Folder ini sekarang harus berisi: icon.svg + icon-192.png + icon-512.png

Fase B — Buat Akun & Repository
B1

Buat Akun GitHub (jika belum punya)

1
Buka browser → pergi ke github.com/signup

Isi email, password, username → verifikasi email yang dikirim

B2

Buat Repository Baru

1
Login GitHub → klik tombol + di pojok kanan atas → New repository

Atau langsung ke: github.com/new

2
Isi form seperti ini:
Repository namemotodash
Visibility✅ Public — wajib agar GitHub Pages gratis bisa dipakai
Initialize README❌ Jangan dicentang
Add .gitignore❌ Jangan dicentang

→ Klik tombol hijau Create repository

3
Setelah dibuat, Anda akan melihat halaman kosong dengan instruksi

Abaikan semua instruksi git clone dll yang muncul di sana — kita tidak akan pakai Git sama sekali.

Fase C — Upload Semua File (Drag & Drop)
C1

Upload Lewat Browser — Drag & Drop

✨ GitHub mendukung drag folder lengkap

Anda bisa men-drag seluruh isi folder (termasuk folder vendor\, css\, js\ yang bersarang) langsung ke kotak upload GitHub — struktur foldernya akan otomatis ikut tersusun rapi.

1
Di halaman repository GitHub Anda → klik Add file → Upload files

Tombolnya ada di kanan atas, dekat tombol hijau "Code"

2
Buka File Explorer di jendela terpisah

Masuk ke folder C:\motodash\ (folder hasil extract)

3
Pilih SEMUA isi folder (bukan folder motodash itu sendiri)

Klik di dalam folder motodash → tekan Ctrl+A → semua file dan folder di dalamnya terpilih (index.html, css, js, vendor, dll)

⚠️

Penting: drag ISI folder motodash (file-filenya), JANGAN drag folder "motodash" itu sendiri. Kalau yang terdrag adalah folder motodash secara utuh, nanti struktur jadi motodash/motodash/index.html yang salah.

4
Drag semua yang terpilih ke kotak upload GitHub
📁 ➡️ ⬆️

Drag file dari File Explorer ke sini

(ini ilustrasi kotak upload di halaman GitHub Anda)

Tunggu proses upload — progress bar muncul untuk setiap file. Untuk 45 file (≈2MB), biasanya selesai dalam 10-30 detik tergantung koneksi internet.

5
Cek semua file sudah terlihat di daftar upload

Scroll area preview di GitHub — pastikan folder vendor, css, js, assets semua kelihatan dengan isinya masing-masing.

6
Scroll ke bawah → isi kotak commit
feat: MotoDash v1.0.0 - upload awal

Lalu klik tombol hijau Commit changes

7
Tunggu sampai selesai → halaman otomatis refresh

Anda akan melihat semua file sudah ada di repository ✅

C2

Jika Drag Sekaligus Gagal/Error

Kadang browser kewalahan kalau terlalu banyak file sekaligus (terutama folder vendor\ yang bersarang dalam-dalam). Kalau itu terjadi, upload per-folder satu-satu:

1
Upload file-file di root dulu

Drag: index.html, manifest.json, sw.js, README.md, CARA_PAKAI.html, dll (yang TIDAK di dalam folder) → Commit

2
Add file → Upload files lagi → drag folder css

→ Commit

3
Ulangi untuk folder js

→ Commit

4
Ulangi untuk folder assets

→ Commit

5
Ulangi untuk folder vendor — ini paling besar, upload sub-foldernya satu-satu jika perlu:

vendor/leaflet → Commit
vendor/leaflet-routing-machine → Commit
vendor/jsmediatags → Commit
vendor/fonts → Commit

💡

Setiap kali drag folder, GitHub otomatis membuat struktur folder yang sama persis di repository. Anda tidak perlu membuat folder secara manual.

Cara Update File di Masa Depan
🔄

Edit / Ganti File Tanpa Aplikasi Lain

GitHub punya editor teks bawaan, jadi Anda bisa edit file langsung di browser:

1
Edit file kecil (misal ganti teks di index.html)

Buka repository → klik nama file → klik ikon ✏️ (pensil) di kanan atas → edit langsung → scroll bawah → Commit changes

2
Ganti file dengan versi baru (misal dari ZIP update)

Add file → Upload files → drag file baru dengan nama yang sama → GitHub otomatis tanya "replace existing file?" → konfirmasi → Commit changes

3
Hapus file yang tidak diperlukan

Klik nama file → klik ikon 🗑️ (tempat sampah) di kanan atas → Commit changes

ℹ️

Setiap "Commit changes" otomatis memperbarui website Anda dalam 1-2 menit (GitHub Pages auto-refresh setiap ada perubahan).

Jika Ada Masalah
🔧

Solusi Masalah Umum

MasalahSolusi
Drag & drop tidak merespon Pastikan browser Chrome/Edge versi terbaru. Coba refresh halaman upload, lalu ulangi drag.
Upload macet di tengah Cek koneksi internet. Refresh halaman, upload ulang folder yang belum selesai.
Folder jadi motodash/motodash/... Anda drag folder motodash utuh, bukan isinya. Hapus folder motodash yang salah di GitHub, ulangi dengan drag ISI folder saja.
Peta tidak muncul setelah deploy Cek folder vendor/ benar-benar terupload lengkap (lihat Step C3). Folder ini sering terlewat saat drag manual.
404 Page Not Found Tunggu 2-5 menit setelah aktifkan Pages. Pastikan branch & folder yang dipilih di Settings → Pages benar (main, root).
Tombol install tidak muncul di Android Pastikan icon-192.png & icon-512.png ada di assets/icons/ dan sudah ter-upload.
Mau ganti banyak file sekaligus Add file → Upload files → drag semua file pengganti (nama sama) → GitHub otomatis replace → Commit changes.
🔍

Cara cek error aplikasi: buka aplikasi di Chrome → tekan F12 → tab Console. Screenshot tulisan merah jika ada, kirim ke Claude untuk bantuan.
